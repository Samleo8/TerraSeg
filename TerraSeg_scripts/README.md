# TerraSeg Scripts


This repository contains the standalone Python scripts for the **TerraSeg** module. TerraSeg is the first self-supervised, domain-agnostic model for LiDAR ground segmentation. It is trained on the [OmniLiDAR](../OmniLiDAR_scripts/) dataset using pseudo-labels generated by [PseudoLabeler](../PseudoLabeler_scripts/), and uses an adapted Point Transformer v3 (PTv3) backbone with dataset-specific normalization disabled. Two released variants are supported: TerraSeg-B (Base, ~46M parameters) and TerraSeg-S (Small, ~12M parameters).

To ensure absolute reproducibility, this project strictly utilizes [uv](https://github.com/astral-sh/uv) as its package manager.


## 🧠 What does TerraSeg do?


TerraSeg performs per-point binary ground / non-ground segmentation of raw LiDAR scans, in real time, across diverse sensor configurations. The training procedure follows the TerraSeg paper and consists of:

1. **Self-supervised supervision:** TerraSeg is supervised by pseudo-labels produced offline by PseudoLabeler on every scan of the OmniLiDAR training corpus.
2. **Domain-agnostic backbone:** A PTv3 backbone with dataset-specific normalization *disabled* (`pdnorm_conditions=None`) and 1D BatchNorm replaced by GroupNorm to stabilize multi-LiDAR training.
3. **3D per-point features:** A constant ones channel, a normalized height channel, and a normalized horizontal-range channel. Raw (x, y, z) coordinates are reserved for PTv3's spatial voxel grid.
4. **Per-scan BCE + symmetric Lovász-Softmax loss** with a dynamic positive-class weight tracked by an exponential moving average (EMA) of the ground / non-ground ratio.
5. **Balanced multi-dataset sampling:** A custom sampler draws scans from each source dataset according to fixed probabilities (paper section A.1), producing a custom epoch length of 51,200 scans (i.e. 100 steps per epoch as batch size is 512).


## ⚙️ Step 1: Prerequisites & setup


1. **Install `uv`** (if you haven't already). For Windows/Mac instructions, see the official `uv` documentation.
```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

2. **Vendor the PTv3 backbone source.** PTv3 is pinned as a workspace member at upstream commit `3229e9b7de1770c8ad17c316f8e349982de509f8`. Follow the one-time vendoring instructions in `ptv3/README.md`. *Note: We already copied the files. This step is only here for reproducibility.*

3. **Sync the Base Environment**:
Navigate to the root directory of the repository (e.g. `TerraSeg/`) and sync the base project environment. This reads the root uv.lock file to instantly build the shared workspace `.venv`.
```bash
cd PUT_YOUR_DIRECTORY_HERE/TerraSeg
uv sync
```


## 🛠️ Step 2: Configure the scripts


Before running the training script, edit the paths near the top of `terraseg_train.py` to point to your local file system:

```python
results_root      = Path("PUT_YOUR_DIRECTORY_HERE/TerraSeg_results")
omnilidar_root    = Path("PUT_YOUR_DIRECTORY_HERE/OmniLiDAR")
pseudolabels_root = Path("PUT_YOUR_DIRECTORY_HERE/PseudoLabeler")
```

*Note 1: Ensure you leave the `.mkdir()` commands intact so the script can build the proper subdirectories.*

*Note 2: The `OmniLiDAR` folder is created using the code in the [OmniLiDAR_scripts](../OmniLiDAR_scripts/) folder; the `PseudoLabeler` folder is created using the code in the [PseudoLabeler_scripts](../PseudoLabeler_scripts/) folder.*

Also update the paths for the evaluation script `terraseg_test.py`:

```python
results_root    = Path("PUT_YOUR_DIRECTORY_HERE/TerraSeg_results")
omnilidar_root  = Path("PUT_YOUR_DIRECTORY_HERE/OmniLiDAR")
checkpoint_path = Path("PUT_YOUR_DIRECTORY_HERE/TerraSeg-B-or-S/best.pth")
```


## 🚀 Step 3: Train TerraSeg


Both released variants share the same training script. To switch between them, set the `VARIANT` constant near the top of `terraseg_train.py` to either:

* `"B"` for the accurate Base model (~46M parameters), or
* `"S"` for the efficient Small model (~12M parameters; halved PTv3 channels and quartered patch sizes).

Then, from the scripts folder, use `uv run` to execute training. Do **not** use the standard `python script.py` command. `uv run` will automatically detect the workspace's virtual environment (which includes the linked `terraseg-utils` and `ptv3`) and execute the script safely.

```bash
cd TerraSeg_scripts
uv run terraseg_train.py
```

This will:
* Build the training dataset from all OmniLiDAR train splits and the three validation datasets (nuScenes, SemanticKITTI, Waymo Perception).
* Train the selected variant for 10 epochs with the published hyperparameters (lr = 1e-3, weight_decay = 5e-3, gradient clipping at 2.5, effective batch size 512 via grad-accumulation, BCE + symmetric Lovász loss, dynamic BCE positive-class weight).
* Log scalars to TensorBoard under `<run_dir>/tb/` and to `<run_dir>/log.txt`.
* Save the best checkpoint to `<run_dir>/best.pth` based on mean validation mIoU across the three eval datasets.

The run directory is `PUT_YOUR_DIRECTORY_HERE/TerraSeg_results/TerraSeg-<VARIANT>/<timestamp>/`, so the two variants can be trained sequentially without overwriting each other.

To monitor training:
```bash
cd TerraSeg_scripts
uv run tensorboard --logdir PUT_YOUR_DIRECTORY_HERE/TerraSeg_results/TerraSeg-<VARIANT>/
```


## 📈 Step 4: Evaluate a trained checkpoint


Set the `VARIANT` constant in `terraseg_test.py` to match the trained checkpoint (`"B"` or `"S"`), and edit `checkpoint_path` to point at the checkpoint you want to evaluate. Then:

```bash
cd TerraSeg_scripts
uv run terraseg_test.py
```

This will compute per-class recall, precision, and IoU on the nuScenes, SemanticKITTI, and Waymo Perception validation splits, plus the mean mIoU across the three benchmarks. Results are printed and saved to `<run_dir>/results.txt`.


## 📂 Repository Structure


**Core Scripts:**
* `terraseg_train.py`: Training entry point for both TerraSeg-B and TerraSeg-S (selected via the `VARIANT` constant).
* `terraseg_test.py`: Evaluation entry point. Loads a checkpoint and reports per-dataset and mean metrics.
* `terraseg_loss.py`: Per-scan BCE + symmetric Lovász loss with a dynamic positive-class weight buffer.
* `terraseg_dataloading.py`: SE(3) augmentation, PTv3 collate function, `BalancedGroupSampler`, and the sampling probabilities from the paper.
* `terraseg_utils.py`: Decision-threshold mIoU sweep and the shared validation loop.


**Related workspace members:**
* `terraseg/` (top-level): the shared TerraSeg library, containing the model definition, the `replace_bn1d_with_gn` BatchNorm-to-GroupNorm swap, the feature-engineering function `compute_terraseg_features`, and the `TerraSegPredictor` class for single-frame deployment inference.
* `TerraSeg_ros2/` (top-level, **not** a uv workspace member): the ROS2 node that wraps `TerraSegPredictor` and exposes TerraSeg as a `sensor_msgs/PointCloud2` -> `sensor_msgs/PointCloud2` filter. Built with `colcon`, not with `uv`.
